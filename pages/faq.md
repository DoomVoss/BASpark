# Frequently Asked Questions (FAQ)

Got a question about BASpark? Check out the common troubleshooting steps below. If your issue persists, feel free to reach out to the community!

---

::: details Q: Modifying parameters has no effect?
**A:** Please make sure you click the **"Apply Changes"** button in the top right corner to push your new settings to the rendering engine.
:::

::: details Q: Prompted with "WebView2 Not Installed" during installation?
**A:** BASpark relies on the Microsoft WebView2 runtime for rendering. Please click the download link in the pop-up prompt, or visit the official Microsoft website to install the Evergreen Bootstrapper manually.
:::

::: details Q: Why are there no effects on certain windows (e.g., Task Manager, Start Menu)?
**A:** Due to Windows security mechanisms (UAC), normal applications cannot overlay high-privilege windows. To fix this, check **"Run as Administrator"** in the settings and restart the software.
:::

::: details Q: How is the system resource consumption?
**A:** In standard test environments, the static standby memory usage is below 100MB. Even during high-frequency clicking or rapid mouse trails, CPU usage fluctuation remains minimal and hardware-friendly.
:::

::: details Q: How do I make the software launch on system startup?
**A:** Simply check **"Launch on Startup"** in the Basic Settings. If the feature doesn't work, check your antivirus or security software to ensure BASpark hasn't been blocked from the startup list.
:::

::: details Q: Cursor effects are offset or misaligned in multi-monitor setups?
**A:** BASpark supports automatic DPI scaling adaptation. If a layout misalignment occurs after plugging/unplugging monitors, please click **"Refresh Monitors"** or restart the software to recalibrate.
:::

::: details Q: Nothing happens when clicking the "Check for Updates" button?
**A:** Please check your internet connection. Server announcements and updates require network access. If your connection is restricted or behind a strict firewall, consider switching your **Network Source** in settings, or check our GitHub repository manually.
:::
